[b]Neuro Companion[/b]

Neuro Companion adds a summon companion you can control with Terraria chat commands.

The mod works as a normal standalone Terraria mod. Randy/Neuro SDK support is optional and only needed if you want to test Neuro/Randy control.

[b]Getting Started[/b]

[list=1]
[*]Craft or obtain the Neuro Companion Staff.
[*]Use the staff to summon Neuro.
[*]Open your inventory and place a magic weapon into Neuro's weapon slot, or use a weapon command.
[*]Use /neuro commands to tell Neuro what to do.
[/list]

Neuro uses her own equipped magic weapon. She does not use the item currently in your hand.

[b]Useful Commands[/b]

[list]
[*]/neuro help - show available commands
[*]/neuro status - show Neuro's current state
[*]/neuro follow - return to follow mode
[*]/neuro attack - attack once
[*]/neuro autoattack 30 - attack nearby enemies for 30 seconds
[*]/neuro attack player - Evil Neuro attacks you
[*]/neuro buff ironskin - apply an allowed buff
[*]/neuro debuff player poisoned - apply an allowed debuff to yourself
[*]/neuro debuff enemy poisoned - debuff a nearby enemy
[/list]

[b]Weapon Commands[/b]

[list]
[*]/neuro weapon status - show Neuro's equipped weapon
[*]/neuro weapon inspect - inspect the weapon you are holding
[*]/neuro weapon set - move the held weapon into Neuro's weapon slot
[*]/neuro weapon take - let Neuro take the strongest valid magic weapon from your inventory
[*]/neuro weapon return - return Neuro's equipped weapon to you
[/list]

[b]Weapon System[/b]

Neuro can use many magic weapons, including:

[list]
[*]Normal magic projectile weapons.
[*]Controllable projectile weapons such as Magic Missile, Flamelash, and Rainbow Rod.
[*]Weapons that place attacks at a target position, such as Resonance Scepter.
[*]Special magic weapons with custom Neuro firing profiles.
[/list]

[b]Staff Upgrades[/b]

The staff tier controls how often Neuro can fire.

[list]
[*]Neuro Companion Staff: basic fire rate
[*]Neuro Companion Staff Mk2: faster fire rate
[*]Neuro Companion Staff Mk3: very fast staff fire limit, still limited by the equipped weapon's own use time
[*]Neuro Companion Staff Mk4: Mk3 fire rate, plus enemy detection through blocks and Neuro-fired projectiles passing through blocks
[/list]

Actual Neuro fire interval is based on both the staff tier and the equipped weapon.

[b]Evil Neuro[/b]

Use:

[list]
[*]/neuro attack player
[/list]

Evil Neuro will attack you with her equipped magic weapon. If she has no valid weapon, she uses a fallback evil bolt.

[b]Optional Randy / Neuro SDK Setup[/b]

Randy integration is optional. You do not need Randy to use the mod normally.

This setup is for testing Neuro/Randy control through websocket.

[b]Requirements[/b]

[list]
[*]tModLoader
[*]Git
[*]Node.js / npm
[*]The Neuro SDK repository
[/list]

[b]1. Clone the Neuro SDK[/b]

Open a terminal somewhere you keep source code, then run:

[code]
git clone https://github.com/VedalAI/neuro-sdk.git
cd neuro-sdk
[/code]

[b]2. Start Randy[/b]

Go into the Randy folder:

[code]
cd Randy
[/code]

If this is your first time setting it up, install dependencies:

[code]
npm install
[/code]

Then start Randy:

[code]
npm start
[/code]

Leave this terminal open.

You should see Randy start running in the terminal. The mod expects Randy's websocket at:

[code]
ws://localhost:8000
[/code]

You can change the websocket URL in the mod config if needed.

[b]3. Open tModLoader[/b]

[list=1]
[*]Enable Neuro Companion.
[*]Enter a world.
[*]Summon Neuro with the Neuro Companion Staff.
[*]Give Neuro a valid magic weapon.
[/list]

Then type this in Terraria chat:

[code]
/neurows connect
[/code]

Check the connection:

[code]
/neurows status
[/code]

You can also send the current game context manually:

[code]
/neurows context
[/code]

If the connection works, you should see connection/activity messages in Terraria and Randy's terminal.

[b]4. Test a Randy Action Manually[/b]

Open a second terminal. This is separate from the terminal running Randy.

Example PowerShell command:

[code]
$body = @{
command = "action"
data = @{
id = "autoattack-test-1"
name = "autoattack"
data = '{"duration_seconds":10}'
}
} | ConvertTo-Json -Depth 5

Invoke-RestMethod `    -Method Post`
-Uri "http://localhost:1337/" `    -ContentType "application/json"`
-Body $body
[/code]

Expected result:

[list]
[*]Randy receives the action request.
[*]The mod receives the action through the websocket connection.
[*]Neuro starts autoattacking in Terraria for about 10 seconds.
[/list]

Another simple test:

[code]
$body = @{
command = "action"
data = @{
id = "evil-neuro-test-1"
name = "attack_player"
}
} | ConvertTo-Json -Depth 5

Invoke-RestMethod `    -Method Post`
-Uri "http://localhost:1337/" `    -ContentType "application/json"`
-Body $body
[/code]

Expected result:

[list]
[*]Evil Neuro attacks the player in-game.
[*]The action result appears in Terraria/debug output.
[/list]

More Randy test commands:

https://docs.google.com/document/d/1ZtHm9jQQ9Ov3Ffl4zOgVQDNZxE1mhyzk/edit?usp=sharing&ouid=107349745517509779788&rtpof=true&sd=true
[/code]

[b]Randy Safety Notes[/b]

Randy/Neuro can only choose high-level actions exposed by the mod.

Randy does not directly control the player, press movement keys, hold attack, use the mouse, or take over normal Terraria controls.

[b]Websocket Commands[/b]

[list]
[*]/neurows connect - connect to Randy
[*]/neurows status - show connection status
[*]/neurows context - send current Terraria context
[*]/neurows disconnect - disconnect from Randy
[*]/neurows help - show websocket command help
[/list]

[b]Troubleshooting Randy[/b]

If Randy does not connect:

[list]
[*]Make sure Randy is running.
[*]Make sure the Randy terminal did not show an error.
[*]Make sure the mod config websocket URL is ws://localhost:8000 unless you changed Randy's port.
[*]Use /neurows status in Terraria.
[*]Use /neurodebug for more detailed mod-side debug info.
[/list]
